# 空库生产验收清单

本清单用于证明发布包可以从没有数据库和业务文件的状态建立可用系统。必须使用临时目录，不能指向现有生产或开发数据。

## 1. 准备

1. 运行全部质量门禁并生成生产构建。
2. 创建独立临时根目录，配置全新的 `DATABASE_PATH`、`UPLOAD_ROOT`、`QUARANTINE_ROOT`、`EXPORT_ROOT`。
3. 设置 `NODE_ENV=production`、回环 `HOST`、独立 `PORT` 和至少 12 字符的临时管理员密码。
4. 确认临时数据库路径不存在，四个目录不包含旧业务文件。

## 2. 初始化与启动

- 运行管理员初始化命令，确认创建成功。
- 再次运行同名初始化，必须拒绝覆盖。
- 启动生产 API，确认迁移表版本与代码当前版本一致、`/api/health` 返回成功。
- 通过反向代理站点登录，确认会话 Cookie 为 HttpOnly、SameSite=Lax、生产环境 Secure。

## 3. 核心业务旅程

按顺序执行并保留截图、请求 ID 或自动化输出：

1. 创建研究项目、中心、访视和项目管理员。
2. 创建筛选表单，发布 v1。
3. 创建筛选记录、保存动态表单、人工判定符合并办理入组。
4. 配置随机化方案，运行模拟、启用并完成一次随机化。
5. 创建随访表单，录入草稿、提交、修改并查看审计差异。
6. 发布兼容的新表单版本，确认旧记录迁移且旧值保留。
7. 创建重复表单记录，上传、下载并删除文件，确认磁盘和数据库一致。
8. 生成 CSV 与 Excel 导出，下载并重新读取工作簿。
9. 查看仪表盘、时间线和审计日志。
10. 建立第二项目与第二中心，证明项目/中心用户无法通过修改 URL 或对象 ID 越权。

## 4. 跨设备与界面

- 1920、1440、1024、768、375px 以及手机横屏无非预期横向滚动。
- 768–1366px 平板类别覆盖横竖屏和触控/键鼠混合输入，不绑定具体型号。
- 中文/英文、浅色/深色均检查；研究自定义标签保持原文。
- 手机只开放筛选、受试者查询、数据录入、随访和个人设置。
- 键盘焦点、错误摘要、44×44px 触控目标、动态字号和 reduced-motion 通过。

## 5. 重启、恢复与清理

- 在存在迁移/导出任务时重启服务，确认任务恢复且不重复完成。
- 停止服务做完整冷备份，恢复到另一临时目录并重复登录、文件下载、随机化结果和审计检查。
- 验收结束后停止服务；只有确认所有路径都属于临时根目录后才能清理。

完成证据应记录在 `REQUIREMENTS_TRACEABILITY.md`，仅有文档步骤而未实际运行不能标记为空库验收完成。
