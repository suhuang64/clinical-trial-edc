# 临床试验 EDC 设计系统

> 项目全局视觉与交互基线。页面级规则放在 `pages/<page-name>.md`，并优先于本文件。

## 1. 设计方向

- 产品类型：多项目、多中心临床试验 EDC 管理后台。
- 风格：医疗专业、可信、克制、高信息密度；不使用营销页式超大标题。
- 设计参数：变化度 2/10、动效度 2/10、桌面密度 8/10、移动密度 5/10。
- 原则：数据正确性优先于装饰；状态必须同时用文字/图标表达，不能只靠颜色。
- 产品运行边界：SQLite 是系统唯一数据库；涉及数据库状态、维护或错误反馈的界面均按单 API 实例与 SQLite 运行语义设计。
- 禁止：拟物/新拟态、玻璃拟态、彩色渐变背景、emoji 图标、装饰性动效。

## 3. 字体排印

- 字体：`Inter, "PingFang SC", "Microsoft YaHei", "Noto Sans CJK SC", system-ui, sans-serif`。
- 不依赖在线字体，确保云服务器或医院网络受限时稳定显示。
- 数字列使用 `font-variant-numeric: tabular-nums`。
- 桌面端：正文 14px/1.5，输入框 14px，页面标题 24px/600，区块标题 18px/600。
- 移动端：正文与输入框至少 16px，页面标题 20px/600，辅助文字不低于 13px。

## 4. 间距、形状与层级

- 4px 基础网格；常用间距 4/8/12/16/24/32px。
- 桌面端表单/表格强调密度；移动端控件间距放宽，避免误触。
- 圆角：输入框 6px、按钮 6px、卡片 8px、对话框 12px。
- 阴影仅用于浮层：卡片默认使用边框，不使用大面积投影。
- 图标统一使用 Element Plus Icons 或 Vant Icons 的同一视觉层级；禁止 emoji。

## 5. 组件与交互

- 每屏仅一个主按钮；删除、退出、作废等危险操作与主操作空间隔离。
- 桌面端控件常规高度 36px；移动端触控区域至少 44×44px。
- 表单始终显示 label；必填标记、帮助文本和单位不可只放 placeholder。
- 普通字段失焦校验；提交时顶部显示错误摘要，并聚焦第一个错误字段。
- 超过 300ms 显示加载反馈，超过 1s 使用骨架屏；提交期间按钮禁用。
- 长表单支持保存草稿；离开未保存页面必须二次确认。
- 动效只用于状态过渡，150–200ms；遵循 `prefers-reduced-motion`。
- 键盘焦点环始终可见；图标按钮必须有可访问名称与 tooltip。

## 6. 响应式规则

- 断点验证：375、768、1024、1440、1920px。
- 桌面端 ≥1024px：固定/可折叠侧边栏、顶部项目上下文、内容区。
- 平板端以设备类别而非具体机型适配，重点覆盖 768–1366px CSS 视口、横竖屏和触控操作；根据可用宽度折叠侧栏，表单采用双列或单列。
- 移动端 <768px：顶部标题栏 + 最多 5 项底部导航；列表转卡片，不横向挤压桌面表格。
- 手机仅承载筛选、受试者查询、数据录入、随访和个人设置，不提供复杂配置器。
- 固定底栏必须包含 safe-area inset，内容不能被导航遮挡。

## 7. 数据可视化

- 趋势使用折线图，中心对比使用横向条形图，比例最多 5 类时才使用环形图。
- 每张图表提供数值 tooltip、图例和表格替代视图。
- 红/绿不能作为唯一差异表达；必须配合标签、形状或图标。
- 图表无数据、加载失败和加载中必须有独立状态。
